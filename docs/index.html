Guide

Download and import

You can download Java Security Essentials here. Then import it in your project as a Java library (.jar).

Hashing

Java Security Essentials allows you to obtain a hash representation of data using either the default algorithm or other available hashing algorithms.

You can use the default hashing algorithm (SHA512) to hash an array of bytes using the Hashing.hash() method.
Note that you have to convert your data (text, numbers, objects etc) into an array of bytes before hashing it:

final String input = "MyRandomInput";
System.out.println(Hashing.hash(input.getBytes()));

The above code outputs:
9bf4dfa7b45625e5475b5d8a6ee4d79c2e97cc1fe6dc88c1345fdd9e8df6167082e491425628bee18ed8862bff5a9006f6ffba909a1f6c658d3f81de35c72460

Hashing algorithms

You can also use other available hashing algorithms listed below (HashType), using the same function but providing the algorithm first:

  • MD5
  • SHA1
  • SHA256
  • SHA384
  • SHA512 (default)
System.out.println(Hashing.hash(HashType.MD5, input.getBytes()));
Hashing with salt

To make your hashes more secure, you need to use salt. Salt provides randomness to the algorithms, making the resulting hash almost impossible to guess or find using a rainbow table. A salt is just a random string of bytes that randomizes the output of the algorithm. Using salt, you need to create it and store it securely so that you can use it to re-create the hash if needed. To do this, you first need to create the salt using the Hashing.salt() method: 

final byte[] salt = Hashing.salt();

Then, you can pass the salt (pun intended) as a parameter to the hash() method:

System.out.println(Hashing.hash(input.getBytes(), salt));
//OR
System.out.println(Hashing.hash(HashType.SHA512, input.getBytes(), salt));

Remember to store the salt in a file or database, otherwise you will not be able to re-create the same hash.
